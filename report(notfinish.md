# 1.问题描述  

## 1.1 项目链接  
[Kamisaki/self_exercise](https://gitee.com/SpaceX233/self_exercise)  

## 1.2 题目内容与要求  
设计并实现一个足球联赛积分管理系统，具体要求如下：  

1. 10 支足球队进行 9 轮循环赛（每轮 5 场，共 45 场）。  
2. 胜一场得 3 分，平一场得 1 分，负一场得 0 分。  
3. 当积分相同时，先比净胜球（进球数 – 失球数），净胜球再相同时再比总进球数。  
4. 程序应能输入每轮比赛的比分，实时统计并显示积分榜，最后将所有比赛记录保存到文件。  

## 1.3 需求分析  

本系统需要实现以下功能：  

**（1）球队管理**  
- 初始化 10 支球队（默认名称 “球队1”~“球队10”）。  
- 用户输入自定义队伍名称，检测名称是否重复。  
- 显示球队编号供比赛录入使用。  

**（2）比赛管理**  
- 录入单场比赛比分（支持按编号或队伍名称两种方式输入）。  
- 检测球队编号/队伍名称是否有效（1~10）。  
- 检测是否出现同一支球队与自己比赛。  
- 检测两支球队是否已经在历史比赛中交手过（防止重复比赛）。  
- 检测进球数是否为负数。  
- 检测某支球队是否在本轮已经出场过（防止同轮一支球队出现两次）。  

**（3）积分计算**  
- 根据比赛结果计算积分（胜 3 分、平 1 分、负 0 分）。  
- 统计胜/平/负场次。  
- 统计进球数、失球数、净胜球。  

**（4）排名展示**  
- 每轮比赛结束后显示当前积分榜。  
- 排名规则：积分 > 净胜球 > 进球（均为降序）。  
- 显示时采用“映射排序”技术，仅影响显示顺序，不改变原始球队数组，从而保证输入编号的稳定性。  

**（5）数据持久化**  
- 将所有比赛记录保存到文件 `matches.txt`。  
- 文件格式清晰，便于查看。

# 2. 概要设计  

## 2.1 模块划分  

本系统共划分为 5 个模块（每个模块对应一个或多个函数）：  

| 序号 | 模块名称 | 模块作用 | 包含函数名 | 源文件名称 |
|------|----------|----------|------------|------------|
| 1 | 球队管理模块 | 球队初始化、队伍名称输入、显示编号 | `init_teams`、`input_team_names`、`check_name_exists`、`show_team_list` | `team.c` |
| 2 | 比赛管理模块 | 比赛录入、输入校验、重复比赛及同轮重复检测 | `input_one_match`、`play_one_round`、`is_match_exists`、`is_team_in_round` | `team.c` |
| 3 | 数据处理模块 | 根据比分更新球队数据（积分计算） | `team_data` | `team.c` |
| 4 | 排序显示模块 | 生成积分榜（仅影响显示，不改变原始顺序） | `show_rank` | `team.c` |
| 5 | 文件存储模块 | 将比赛记录写入文件 | `save_matches` | `team.c` |

## 2.2 文件结构  

- `team.h`：头文件，包含结构体定义与所有函数原型声明。  
- `team.c`：源文件，包含所有函数的具体实现。  
- `main.c`：主程序文件，负责程序入口、流程控制以及函数调用。  

## 2.3 系统主要执行逻辑设计  

```
开始
  ↓
init_teams()                 // 初始化 10 支球队（默认名称）
  ↓
input_team_names()           // 用户输入自定义队伍名称（检测重复）
```

# 3. 详细设计  

## 3.1 用户交互界面设计  

- **欢迎界面**：程序启动时一次性打印系统名称与比赛规则，帮助用户了解输入格式与限制。  
- **队伍名称输入界面**：逐个提示 “第X支：”，用户输入名称后进行非空、长度、重复检测；错误时给出明确提示并要求重新输入。  
- **球队编号显示界面**：输入完所有队伍名称后，显示所有球队的编号（1~10）及对应名称，供后续比赛录入参考。  
- **比赛录入界面**：每轮每场提示 “第X场：输入 球队A编号 球队B编号 比分x:y：”。支持两种输入方式：  
  - 按编号输入，如 “1 2 3:1”  
  - 按队伍名称输入，如 “巴西 阿根廷 3:1”（程序内部会通过名称查找对应编号）  
  错误信息包括：格式错误、编号无效、同队比赛、进球为负、重复比赛、本轮已出场等。  
- **积分榜显示界面**：每轮比赛结束后，程序调用 `show_rank` 以表格形式显示当前积分榜，表头对齐，包含排名、队伍编号、队名、积分、胜、平、负、进球、失球、净胜球。  
- **最终积分榜与保存**：所有轮次结束后，再次显示最终积分榜，随后调用 `save_matches` 将所有比赛记录写入 `matches.txt`，并在控制台输出 “比分已保存到 matches.txt”。  

## 3.2 核心数据结构设计  

```c
struct Team {
    char name[20];       // 队伍名称
    int score;           // 积分
    int win;             // 胜场
    int draw;            // 平场
    int lose;            // 负场
    int goal;            // 进球
    int lose_goal;       // 失球
    int more_goal;       // 净胜球
};

struct Match {
    int round;           // 轮次
    char a_name[20];     // A 队名称
    char b_name[20];     // B 队名称
    int a_goal;          // A 队进球
    int b_goal;          // B 队进球
};
```

- `Team` 数组长度固定为 `N = 10`（十支球队）。  
- `Match` 数组长度固定为 `MAX_MATCHES = 45`（九轮五场）。  
- 这些结构体是系统的核心数据结构，保存了所有必要的统计信息。  

## 3.3 核心算法设计  

### 3.3.1 积分计算算法（`team_data`）  

**功能描述**：根据两支球队的进球数更新各自的积分、胜/平/负场次、进球数、失球数、净胜球。  

**输入**：  
- `a`：A 队在 `Team` 数组中的下标（0‑based）  
- `b`：B 队的下标  
- `x`：A 队进球数  
- `y`：B 队进球数  

**输出**：对应 `Team` 结构体的字段被直接修改。  

**算法流程**：  
1. 更新进球与失球：  
   - `t[a].goal += x;` `t[a].lose_goal += y;`  
   - `t[b].goal += y;` `t[b].lose_goal += x;`  
2. 计算净胜球：  
   - `t[a].more_goal = t[a].goal - t[a].lose_goal;`  
   - `t[b].more_goal = t[b].goal - t[b].lose_goal;`  
3. 根据胜负平更新积分与胜/平/负场次：  
   - 若 `x > y`：A 胜 → `t[a].score += 3; t[a].win++; t[b].lose++;`  
   - 若 `x < y`：B 胜 → `t[b].score += 3; t[b].win++; t[a].lose++;`  
   - 若 `x == y`：平局 → `t[a].score++; t[b].score++; t[a].draw++; t[b].draw++;`  

### 3.3.2 队伍名称重复检测（`check_name_exists`）  

遍历已输入的队伍名称（下标 `0 … current‑1`），若发现 `strcmp(t[i].name, name) == 0` 则返回 1（表示重复），否则返回 0。  

### 3.3.3 防止重复比赛（`is_match_exists`）  

遍历已保存的比赛记录（下标 `0 … count‑1`），若出现以下情况之一则返回 1（表示已比赛）：  
- `strcmp(m[i].a_name, a_name) == 0 && strcmp(m[i].b_name, b_name) == 0`  
- `strcmp(m[i].a_name, b_name) == 0 && strcmp(m[i].b_name, a_name) == 0`  

否则返回 0。  

### 3.3.4 防止同轮一支球队出现两次（`is_team_in_round`）  

遍历已保存的比赛记录，记录所在轮次等于当前轮次且（`strcmp(m[i].a_name, team_name) == 0 || strcmp(m[i].b_name, team_name) == 0`）时返回 1，否则返回 0。  

### 3.3.5 显示积分榜（`show_rank`）  

**目标**：在不改变原始 `Team` 数组顺序的前提下，按积分 > 净胜球 > 进球（均降序）显示排名。  

**算法步骤**：  
1. 创建一个整型数组 `rank_idx[N]`，初始化为 `0, 1, …, N‑1`（表示原始下标）。  
2. 对 `rank_idx` 进行冒泡排序：比较两个相邻元素对应的球队。  
   - 取 `idx1 = rank_idx[j]`，`idx2 = rank_idx[j+1]`。  
   - 比较 `t[idx1].score` 与 `t[idx2].score`：  
     - 若前者小于后者 → 需要交换。  
     - 若积分相等，再比较 `t[idx1].more_goal` 与 `t[idx2].more_goal`：  
       - 若前者小于后者 → 需要交换。  
       - 若净胜球也相等，可再比较进球数（按需求可加）。  
   - 若需要交换，则交换 `rank_idx[j]` 与 `rank_idx[j+1]`。  
3. 排序完成后，依据 `rank_idx` 的顺序打印排名：  
   - 排名 = `i + 1`（`i` 为 `rank_idx` 下标）  
   - 队伍编号 = `rank_idx[i] + 1`  
   - 队伍名称 = `t[rank_idx[i]].name`  
   - 其余字段直接取自 `t[rank_idx[i]]`。  

这样原始 `Team t[ ]` 的顺序未被改动，后续输入仍然按照原来的编号（1 对应 t[0]，2 对应 t[1]……）正确对应球队。  

### 3.3.6 文件存储（`save_matches`）  

以文本方式打开（或创建）文件 `matches.txt`，写入表头后遍历所有比赛记录，每行输出：  
`%-4d %-12s %-12s %6d %6d\n`（轮次、A 队名称、B 队名称、A 进球、B 进球），随后关闭文件并给出成功提示。

# 5. 总结  

## 5.1 遇到的问题及解决方法  

| 问题 | 解决方法 |
|------|----------|
| **程序可能因错误输入崩溃或死循环** | 添加完善的输入校验，使用 while(1) 循环配合 continue 直到输入正确。 |
| **队伍名称重复检测漏掉** | 通过遍历已输入队伍数组 `check_name_exists` 实现精确重复判断。 |
| **同轮一支球队出现两次** | 新增 `is_team_in_round` 函数，在每场比赛校验前检查本轮是否已出场。 |
| **两支球队重复比赛** | 通过 `is_match_exists` 检查历史比赛记录。 |
| **排序会改动输入编号对应关系** | 在 `show_rank` 内部使用数组 `rank_idx` 进行排序，仅影响显示顺序，不改动原始 `Team` 数组。 |
| **文件保存路径或权限问题** | 在 `fopen` 失败时给出明确提示 “保存失败！” 并返回，不影响后续流程。 |
| **自己可以和自己比赛** | 添加if判断两支队伍是否相同。 |
| **格式输出不美观** | 使用 printf 的格式化输出，如 `%-10s` 实现左对齐。 |
