# 1.问题描述

## 1.1项目链接

[Kamisaki/self_exercise](https://gitee.com/SpaceX233/self_exercise)

## 1.2题目内容与要求

设计并实现一个足球联赛积分管理系统，**具体要求**如下：

​       10个足球队进行9轮循环赛，胜一场积3分，平一场积1分，输一场积0分，如果两个队积分相同，进行净胜球的比较，如果净胜球再相同进行总进球的比较。编写一程序输入每轮比赛的比分，并可以存储。最后统计出每轮比赛的积分榜。

## 1.3需求分析

本系统需要实现以下功能：

**（1）球队管理**
- 初始化10支球队
- 用户输入自定义队名
- 检测队名是否重复
- 显示球队编号供比赛录入使用

**（2）比赛管理**
- 录入单场比赛比分
- 支持按编号或队名两种方式输入
- 检测球队编号/队名是否有效
- 检测是否自己与自己比赛
- 检测两支球队是否重复比赛
- 检测进球数是否为负数

**（3）积分计算**
- 根据比赛结果计算积分（胜3分、平1分、负0分）
- 统计胜平负场次
- 统计进球数、失球数、净胜球

**（4）排名展示**
- 每轮比赛结束后显示当前积分榜
- 按积分、净胜球、进球进行排序

**（5）数据持久化**
- 将所有比赛记录保存到文件
- 文件格式清晰，便于查看

### 2.4.2函数原型设计

**1. 球队管理模块函数**

（1）函数：init_teams

```c
void init_teams(struct Team t[]);
```

函数作用：初始化10支球队，将所有数据清零，并设置默认队名"球队1"到"球队10"。

参数解释：t[] 为球队数组，传入数组名即可。

（2）函数：input_team_names

```c
int input_team_names(struct Team t[]);
```

函数作用：循环接收用户输入的10个队名，每输入一个队名都会调用check_name_exists检查是否重复。

参数解释：t[] 为球队数组，返回值1表示成功，0表示失败。

（3）函数：check_name_exists

```c
int check_name_exists(struct Team t[], char* name, int current);
```

函数作用：遍历已输入的队名，检查新输入的队名是否与已有队名重复。

参数解释：
- t[]：球队数组
- name：要检查的队名
- current：已输入的队名个数
- 返回值1表示重复，0表示不重复

（4）函数：show_team_list

```c
void show_team_list(struct Team t[]);
```

函数作用：按编号顺序显示所有球队名称，供用户比赛录入时参考。

参数解释：t[] 为球队数组。

**2. 比赛管理模块函数**

（1）函数：input_one_match

```c
int input_one_match(struct Team t[], struct Match m[], int* match_cnt, int round, int match_num);
```

函数作用：接收用户输入的一场比赛信息，包含队名/编号和比分。函数内部进行了多项输入校验，确保数据合法。

参数解释：
- t[]：球队数组
- m[]：比赛记录数组
- match_cnt：当前已记录的比赛场数（指针）
- round：当前轮次
- match_num：当前是第几场比赛
- 返回值1表示成功

（2）函数：play_one_round

```c
void play_one_round(struct Team t[], struct Match m[], int* match_cnt, int round);
```

函数作用：完成一轮5场比赛的录入，每场比赛调用input_one_match，5场结束后调用sort_team排序并调用show_rank显示积分榜。

参数解释：
- t[]：球队数组
- m[]：比赛记录数组
- match_cnt：比赛计数
- round：当前轮次

（3）函数：is_match_exists

```c
int is_match_exists(struct Match m[], int count, char* a_name, char* b_name);
```

函数作用：检查两支球队是否已经比赛过，防止重复比赛。

参数解释：
- m[]：比赛记录数组
- count：已记录的比赛场数
- a_name：A队名称
- b_name：B队名称
- 返回值1表示已存在比赛，0表示未比赛

**3. 数据处理模块函数**

（1）函数：team_data

```c
void team_data(struct Team t[], int a, int b, int x, int y);
```

函数作用：根据比赛进球数更新两支球队的数据，包括：积分、胜平负场次、进球数、失球数、净胜球。

参数解释：
- t[]：球队数组
- a：A队下标
- b：B队下标
- x：A队进球数
- y：B队进球数

（2）函数：sort_team

```c
void sort_team(struct Team t[]);
```

函数作用：使用冒泡排序算法对球队按规则排序：积分高者优先；积分相同则净胜球多者优先；积分和净胜球都相同则进球多者优先。

参数解释：t[] 为球队数组。

**4. 输出模块函数**

（1）函数：show_rank

```c
void show_rank(struct Team t[]);
```

函数作用：以表格形式显示当前积分榜，包含：排名、队名、积分、胜、平、负、进球、失球、净胜球。

参数解释：t[] 为球队数组。

（2）函数：save_matches

```c
void save_matches(struct Match m[], int cnt);
```

函数作用：将所有比赛记录保存到文件matches.txt，文件格式包含轮次、A队、B队、A进球、B进球。

参数解释：
- m[]：比赛记录数组
- cnt：比赛总场数

## 3.2核心数据结构设计

**1. 球队结构体**

```c
struct Team {
    char name[20];       // 球队名称
    int score;          // 积分
    int win;            // 胜场
    int draw;           // 平场
    int lose;           // 负场
    int goal;           // 进球
    int lose_goal;      // 失球
    int more_goal;      // 净胜球

```

该结构体是本系统的核心数据结构，包含了球队的所有统计信息。

**2. 比赛记录结构体**

```c
struct Match {

    int a_goal;          // A队进球
    int b_goal;          // B队进球
};
```

该结构体用于保存每场比赛的记录，便于赛后查询和文件存储。

（2）输入数据

- a：A队下标
- b：B队下标
- x：A队进球数
- y：B队进球数

（3）输出数据

更新两支球队的score、win/draw/lose、goal、lose_goal、more_goal字段。

# 5.总结

## 5.1遇到的问题及解决方法

| 问题 | 解决方法 |
|------|----------|
| 输入含有空格时只读到第一个单词 | 改用 `fgets` 读取整行输入，而不是使用 `scanf("%s")` |
| 队名输入后直接开始比赛会吞掉换行符导致格式错误 | 在两段输入之间添加 `while (getchar() != '\n')` 清空缓冲区 |
| 自己可以和自己比赛，两支球队可以重复比赛 | 添加if判断两支队伍是否相同，添加 `is_match_exists` 函数，遍历已有比赛记录进行检查 |
| 程序可能因错误输入崩溃或死循环 | 添加完善的输入校验，使用 while(1) 循环配合 continue 直到输入正确 |
| 格式输出不美观 | 使用 printf 的格式化输出，如 `%-10s` 实现左对齐 |
